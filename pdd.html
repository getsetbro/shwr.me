Prototype Driven Development

PDD

With Prototype Driven Development UI components are tested in a prototype tool (like PatternLab link) to insure that they will work in all widths.

Objectives

  1. Early UI Testing / Feedback
  2. Set patterns to make it easy for integration team
  3. Most valuable for large projects with separate front-end and back-end teams
  4. Consistent methods and conventions

Caveats

  1. This does not replace axure / axshare
  2. Guaging interest

Contents

  1. Prototyping with PatternLab
  2. Visual Studio solution setup with open source utilities (NodeJS, NPM, Grunt, Gulp)
  3. Patterns for modular JS
  4. CSS conventions for reusable components

PatternLab

With PatternLab you build components in their smallest re-usable form and then add them together. For instance a "Single comment" is part of a thread of comments.
